# Decisions

This folder holds architecture renegotiation notes for the reports workbench.

Create a new decision note here whenever implementation needs to consciously change a locked architectural choice described in:
- `architecture-renegotiation-protocol.md`
- `implementation-principles.md`
- `visual-token-spec.md`
- `state-visibility-matrix.md`
- `review-evidence-protocol.md`

## File Naming

Use:

`YYYY-MM-DD-short-kebab-name.md`

## Minimum Template

```md
# <Decision Title>

- Date:
- Status:

## Context

## Old Decision

## Pressure Or Failure Mode

## Options Considered

## Chosen Decision

## Consequences

## Docs To Update

## Tests Or Review Evidence Impact
```
